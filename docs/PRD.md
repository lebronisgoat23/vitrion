## 1. 產品願景

### 1.1 問題陳述

現代人面臨「精力管理」的根本挑戰：

1. **資訊過載，行動不足** — 知道該運動、該早睡，但就是做不到
2. **被動式健康管理** — 只有生病了才去看醫生，缺乏預防機制
3. **量化數據無感** — 穿戴裝置給你一堆數字，但不知道「現在該做什麼」
4. **通用建議無效** — 一般健康 app 給的是「平均人」的建議，不是「你」的建議

### 1.2 解決方案

**Vitrion = 即時生理感測 + 情境感知推薦 + 遊戲化行為引擎**

```
不是告訴你「什麼是好的」
而是告訴你「現在該做什麼」
並且讓你願意去做
```

### 1.3 獨特價值主張 (UVP)

> **「用手機相機讀懂你的身體，在你疲勞之前主動介入」**

| 競品 | 定位 | VS Vitrion |
|------|------|-----------|
| ChatGPT Health | 健康問答助手 | ❌ 無法感測，只能回答 |
| Apple Health | 數據儀表板 | ❌ 只顯示數據，不引導行動 |
| Oura Ring | 穿戴式追蹤 | ❌ 需要額外硬體，被動報告 |
| Headspace | 冥想引導 | ❌ 單一維度，無整體觀 |

## 2. 目標用戶

### 2.1 初期核心用戶 (Beachhead)

**創業者 / 高壓知識工作者**

| 特徵 | 描述 |
|------|------|
| 年齡 | 25-40 歲 |
| 痛點 | 工作壓力大、睡眠不穩、精力波動影響產出 |
| 行為 | 已有量化習慣（用過穿戴裝置）、願意嘗試新工具 |
| 付費意願 | 高（時間 > 金錢） |
| 獲取管道 | Twitter/X、創業社群、Podcast |

### 2.2 用戶 Persona

**Alex，32 歲，新創公司 CTO**

> 「我每天工作 12 小時，但常常在下午 3 點就開始當機。喝咖啡只是短期解法，我需要知道為什麼會這樣，以及怎麼預防。」

**需求**:
- 了解自己的精力模式
- 在關鍵會議前保持最佳狀態
- 避免咖啡因影響睡眠
- 建立可持續的健康習慣

## 3. 功能規格

### 3.1 臉部生理掃描

#### 功能描述
用戶打開相機，系統自動分析臉部特徵，推估健康狀態。

#### 技術實現
- **輸入**: 前鏡頭即時影像
- **處理**: MediaPipe Face Landmarker (468 個臉部關鍵點)
- **分析指標**:
  - `face_width_height_ratio`: 臉部寬高比（水腫指標）
  - `jawline_sharpness_deg`: 下頷線角度（體脂相關）
  - `chin_angle_deg`: 下巴角度

#### 輸出
```json
{
  "status": "success",
  "data": {
    "scan_type": "FACE",
    "face_width_height_ratio": 0.85,
    "jawline_sharpness_deg": 125.5,
    "bio_age": 28,
    "aging_rate": 0.95,
    "entropy_score": 35,
    "supply_drop": {
      "item_name": "Electrolyte Pack",
      "rarity": "RARE",
      "description": "輕微水腫偵測，建議補充電解質",
      "buff": "+10% Hydration"
    }
  }
}
```

#### 驗收標準
- [ ] 在正常光線下，臉部偵測成功率 > 95%
- [ ] 分析回應時間 < 500ms
- [ ] 掃描結果可重複性 > 80%（同一人連續掃描結果一致）

### 3.2 Founder Protocol (時間感知推薦)

#### 功能描述
根據當前時間和用戶狀態，自動推薦「下一步最佳行動」。

#### 時段邏輯

| 時段 | 時間範圍 | 核心目標 |
|------|----------|---------|
| 早晨 | 05:00 - 11:00 | 啟動皮質醇、設定節律 |
| 中午 | 11:00 - 14:00 | 營養攝取、咖啡因窗口 |
| 下午 | 14:00 - 18:00 | 訓練窗口、維持精力 |
| 傍晚 | 18:00 - 21:00 | 準備入睡、減少刺激 |
| 夜間 | 21:00 - 05:00 | 睡眠優化 |

#### 狀態感知介入
用戶可點選當前狀態（餓、累、無法專注），觸發情境化建議：

- **餓** → 先喝水（可能是口渴）、氣泡水+蘋果醋
- **累** → 出去走、檢查水分攝取（不是直接喝咖啡）
- **無法專注** → 咖啡因+茶氨酸、番茄鐘

#### 驗收標準
- [ ] 時段自動切換正確
- [ ] 狀態介入邏輯與健康科學一致
- [ ] 點擊完成有 haptic 回饋

---

### 3.3 咖啡因計時器

#### 功能描述
追蹤咖啡因攝取，提醒最後安全飲用時間。

#### 核心邏輯
- 咖啡因半衰期：約 5-6 小時
- 睡前體內咖啡因應 < 25mg（影響睡眠的閾值）

#### UI 元素
- 攝取時間輸入
- 劑量選擇（Espresso 60mg / 美式 100mg / 自訂）
- 即時顯示體內濃度曲線
- 倒數計時至「安全入睡時間」

#### 驗收標準
- [ ] 計算邏輯符合藥理學半衰期模型
- [ ] 支援多次攝取疊加
- [ ] 有視覺化濃度曲線

---

### 3.4 遊戲化系統

#### XP 機制
| 行動 | XP 獎勵 |
|------|--------|
| 完成每日掃描 | +50 XP |
| 完成協議行動 | +20 XP |
| 連續 7 天 | +200 XP Bonus |
| 完成 30 天挑戰 | +1000 XP |

#### 等級系統
- 每 500 XP 升一級
- 升級有動畫 + haptic 回饋

#### Streak 機制
- 顯示連續使用天數
- 斷掉後有「恢復」機制（24 小時內可補救）

## 8. 風險與緩解

| 風險 | 影響 | 緩解策略 |
|------|------|---------|
| CV 準確度不足 | 用戶不信任 | 透明化呈現「參考」性質，強調趨勢而非絕對值 |
| 用戶隱私顧慮 | 轉化率低 | 強調本地處理、不上傳原始影像 |
| AI 巨頭競爭 | 被追趕 | 專注垂直場景、累積獨特數據護城河 |
| 行為改變困難 | 流失率高 | 強化遊戲化、社群功能 |
